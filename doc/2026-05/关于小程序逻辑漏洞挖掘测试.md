# 前言  
  
某个老年大学就餐系统小程序，存在多处逻辑漏洞，但是这个小程序不仅存在就餐系统，还存在其他功能点，比如购买商品，一些服务项目，比如针灸按摩啥的，反正小程序功能越多，漏洞也就越多，而且小程序大部分都是可以直接注册登录的，所以测试起来相对容易一点。  
# 优惠卷刷取  
  
访问某小程序，登录个人中心存在一个认证功能点  
  
![](https://mmbiz.qpic.cn/sz_mmbiz_png/ianpxKPnLHoJicnMzHoYwsSREpKAHBLU49BPSeGIMu3MerCpXuFh5ibUktSicNIRjH9acYvn9yzDicpjFBSyP5GJWUm8MggRJJmQvrmOcpWLAJ8U/640?wx_fmt=png&from=appmsg "")  
  
这里进入后百度上任意搜索一个身份证姓名提交申请测试一下  
  
![](https://mmbiz.qpic.cn/sz_mmbiz_png/ianpxKPnLHoIHwuSv2dgpwdtNbpIaK4avVgBiaIzSmxCibHNOK2Uda2dEL8eZn2jwjy4micCxXjkRmYSV025e4EeVk5J33hxdd6libdUbhFSWTgM/640?wx_fmt=png&from=appmsg "")

![](https://mmbiz.qpic.cn/sz_mmbiz_png/ianpxKPnLHoJ7V1qbuXPRdE72IuXNzibLZjBIZV5r0HZ6oKu1FE7UMKxj6nJAmxIpub8smn134ff0Pb113qRbm7mDhibaWoa6CJHI5RtaMKv0g/640?wx_fmt=png&from=appmsg "")

![](https://mmbiz.qpic.cn/mmbiz_png/ianpxKPnLHoK6UoRiaWCMiagG4J8cjbB2ibZp6pzgUbo6KC8X7d1ZK5USlWkJSB1CtQL12hewDpmVic2IceIicibbVgN1TJ4Pl4FK0bA5wwy7K99mw/640?wx_fmt=png&from=appmsg "")

# 地址越权  
  
地址功能点，存在id校验  
  
![](https://mmbiz.qpic.cn/sz_mmbiz_png/ianpxKPnLHoIEoZnj8MKYEEqK4GSqjiblIqczMIDnnuxict4WoNMqBlgSicUHzHRMlhXy1CRPHGqSxibnfR9Ccd3XD6rHTyCt3w1gEvVgvvicD4fU/640?wx_fmt=png&from=appmsg "")  
  
通过更换其他用户的地址id值无法进行越权处理，说明此处是不存在越权的  
  
![](https://mmbiz.qpic.cn/mmbiz_png/ianpxKPnLHoKEkibQZZjYwg0k5kIibzGdjKuI2apDGJYX6hAGPpZlLf2bCpICAqF5nfNZSvQPtHNAf8LQXbiciaE4HF7VkibWhAk6VrNubPmveE98/640?wx_fmt=png&from=appmsg "")

![](https://mmbiz.qpic.cn/mmbiz_png/ianpxKPnLHoKtIGjXbkr8A72D2za3Ad0ia05NFxHg3RrKD9aGEvnSSz1sUOk9SNoLejtEWljYeLOdnTGK1nSGU7SpfAQZ8NcicRibiaZO2siaqGts/640?wx_fmt=png&from=appmsg "")  
  
下单，提交订单处  
  
![](https://mmbiz.qpic.cn/mmbiz_png/ianpxKPnLHoLp2YdhMesVcpicK1TI0FUhDibe2ic04w9RumLc4uOsjmnzSod4ia1PfClzEdgST5lnSzmicqgpKYRqUGJQhRqcibQj1CVSP4GzdsrUA/640?wx_fmt=png&from=appmsg "")

# 支付漏洞  
  
某个小程序，需要登录，但是可以注册用户  
  
![](https://mmbiz.qpic.cn/sz_mmbiz_png/ianpxKPnLHoKHumicuQvSj13aCqibJNiaNYwqmGggw8SibZBmYu8RlygEG5eDXBFicjJDsOwRKdyrpdATM4MG2ovZJ1h50Nn4p2gvsaRu9jlQnJwI/640?wx_fmt=png&from=appmsg "")  
  
这里是存在支付漏洞，每个商品信息有两个控制的，其中gooid代表商品id，"value":"xxx"代表商品价格，其在校验的过程中是只对value进行校验（这里是猜测的，因为数据包参数比较多，所以我在初次更改value值的时候发现价格并没有更改成功）。在加入购物车的时候替换id，不替换value，就可以以低价商品的value购买高价格商品  
  
首先记录高价值商品id：xxxxxxx6946  
  
![](https://mmbiz.qpic.cn/mmbiz_png/ianpxKPnLHoKKZgGwTO52s3O5jGVr06z7KJsCRmVDhDHU6pAR2MdN0cI7gsCE1XNk3PNXTdBzsXuaeHeZo5GKB8C2ibH3sM4ThPx7ibf5jqJ64/640?wx_fmt=png&from=appmsg "")  
  
这里可以发现4块钱是单次的费用  
  
![](https://mmbiz.qpic.cn/mmbiz_png/ianpxKPnLHoIPGD0Tpra5NMKdNsUrpK0jb27K8RYJk1Z5kjgdSZsjvxfVkXicozTg22viaNucqay4eat1QBzfcUIq9Dnlk2qicAMibp00CPmmiads/640?wx_fmt=png&from=appmsg "")  
  
在低价值商品4元单次送餐页面加入购物车，加入的时候替换gooid为高价值商品id值xxxxxxx6946  
  
![](https://mmbiz.qpic.cn/sz_mmbiz_png/ianpxKPnLHoLricE3iaQtOzZicw6qZ0Ls2BGOmxDSIYaKsKeOE3UicjYXZx6fVwjZAYIsTPE5E1nDnaOeu6eFN5tHFHwrxLutf4EicmPK8u0uuR4M/640?wx_fmt=png&from=appmsg "")  
  
这里直接以四块钱单次的费用下单包月的商品价值，从而造成支付漏洞  
  
![](https://mmbiz.qpic.cn/mmbiz_png/ianpxKPnLHoIOiahfmxuAfz5623Nx58KUicFhp5FfONibWNzaLGbXbQz70yjlcRL4hic3eXkPzNKGib55dIh9Sz9Qz8r9Eia5zUpUiaTxfKpHGqIjeI/640?wx_fmt=png&from=appmsg "")  
  
如下图，通过手机扫码可以直接支付  
  
![](https://mmbiz.qpic.cn/sz_mmbiz_png/ianpxKPnLHoJrP5KgucdP664K44gOkMlXRXApytUgtng1tsu6ocmVFlYR8lZ0kRSmzpCibkQPS8u76wiauETTc7ClH5icLtBvZlNf85VA07ufTI/640?wx_fmt=png&from=appmsg "")
