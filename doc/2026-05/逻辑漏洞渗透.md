### GB/T 36627-2018信息安全技术网络安全等级保护测试评估技术指南中，对逻辑漏洞的解释：  
- 业务逻辑缺陷漏洞（Business Logic Vulnerability） 是指由于应用程序的业务逻辑处理存在缺陷（如流程跳跃、状态篡改、权限校验缺失等），导致攻击者能够绕过正常业务流程，实现非预期的操作后果（如越权访问、任意密码重置、低价交易等）。  
  
在工作中，逻辑设计缺陷是由于应用在最初设计时由于未考虑全面，在登录、注册、找回密码、支付模块中程序的判断逻辑及程序的处理流程上存在缺陷，导致攻击者可以绕过程序的处理流程，从而达到特定的目的，如暴力破解密码，任意用户注册、任意用户登录、任意密码重置及各种支付漏洞。

#### 短信轰炸  
  
原理由于短信业务逻辑设计缺陷，没对短信发送次数做限制，导致可以大量重复发送短信验证码。该漏洞会对其他用户造成骚扰或使厂商的运营商短信费用的增加，造成损失。  
  
具体操作：抓取发送短信的数据包到Repeater器中，重复点击send。手机在同一时间内收到大量短信验证码。  
  
会员注册页面可以看到注册需要短信获取验证码：  
  
![](https://mmbiz.qpic.cn/mmbiz_png/Shiamu7mxKIDSybRXiabRJbnGlhNwekSqbby4enbZOicgv262vJibS3VzQwTNHSq7cPGCiapsH3wM2UE24NIGMtAcs6icb9ribuGqAicmlFMtktwy8k/640?wx_fmt=png&from=appmsg "")  
  
漏洞复现  
  
第一步：进入会员注册页面，输入手机号，获取验证码，测试短信接口正常  
  
![](https://mmbiz.qpic.cn/mmbiz_png/Shiamu7mxKICpiaJhKUDRB3bvcEjBHuGnA5CPibt8m2VyYgH1PibELIgIQibe5mc4mfZN6f1rjLibbuwCPd4Pfd8iaTmHPFIYJMqadu0ibjOPEWOvCw/640?wx_fmt=png&from=appmsg "")  
  
第二步：注册，打开BP进行抓包，查看数据包，可以看到短信验证码也是直接存在cookie中的：  
  
![](https://mmbiz.qpic.cn/sz_mmbiz_png/Shiamu7mxKID7YLHuF6Xqf3FtF1CqhfpestEViaR9pnJI2PsGs9LlSNA3KgSPRRZ3IuQdzoNq4ibxiaY8KBF0LvpGAwm5nUKsFht8NAQ5XVLjIY/640?wx_fmt=png&from=appmsg "")  
  
此时如果该接口存在安全漏洞，比如没有限制短信发送次数和时间间隔限制、那么我们就可以将以上数据包发到repeater，然后不断地重放请求数据包，然后就会不断地产生新的短信验证码通过接口发送至指定的手机，造成短信轰炸的现象，也就是短信炸弹。  
修改手机号接收验证码  
  
用自己的手机收到正确验证码，在点击注册时拦截包将手机号改为其他手机号，如果成功的话就注册了别人的手机号，这是因为后端仅验证了验证码是否是正确的而没有验证验证码是否与手机匹配。  
  
**三、常见的逻辑漏洞—登陆**  
  
  
![](https://mmbiz.qpic.cn/mmbiz_gif/Shiamu7mxKIC6e8e0lygPEVGHsUibo2icpvgCDaKmZ7ic2ic3zDgtSV3NXFvCuMCc2JYoFoweVkuClpEghUxwYxujWks1HTULe2VHdILesrl9qico/640?wx_fmt=gif&from=appmsg "")  
  
  
  
登录处可能存在任意用户登录、验证码可绕过、用户账号可撞库等问题。  
  
短信轰炸  
  
测试短信轰炸与注册处测试步骤一样（一般来说注册处有短信轰炸的话这里也会有）。  
  
验证码爆破  
  
至于手机验证码，通常是尝试爆破，如果网站发到手机上的短信没有写什么在xx时间内有效之类的则有可能没有时间限制，将登录包右键发送至Intruder（即测试器模块）设置好爆破位置后在载荷里选择数值后开始爆破。主要原因对验证码有效期和请求次数没有进行限制。  
  
漏洞复现  
  
semcms靶场的邮箱认证码默认是四位数字组成，可以使用burp  
suite进行爆破，密码随便输入(本实验输入1  
2345)  
，认证码也随便输入1  
-  
4位数字，然后通过burpsuite进行抓包，点击确认找回。  
  
![](https://mmbiz.qpic.cn/sz_mmbiz_png/Shiamu7mxKIAjnwm4ZicEJxSHuls9m8D1SMgTeayxuYNLR0DPvFkCBRzSBusUhv1RN5YXhvXGmu6cphibOuumbDJbYM5gibYVJh0SeLFBC5ZrU4/640?wx_fmt=png&from=appmsg "")  
  
修改用户参数  
  
查看数据包中是否有用户id之类的参数，抓包修改参数，验证系统是否拥有完善权限管理和鉴权机制。  
  
**四、常见的逻辑漏洞—密码找回**  
  
  
![](https://mmbiz.qpic.cn/mmbiz_gif/Shiamu7mxKIC6e8e0lygPEVGHsUibo2icpvgCDaKmZ7ic2ic3zDgtSV3NXFvCuMCc2JYoFoweVkuClpEghUxwYxujWks1HTULe2VHdILesrl9qico/640?wx_fmt=gif&from=appmsg "")  
  
  
  
导致密码找回功能的逻辑漏洞的原因大致有以下几类：  
- 验证码爆破的，对验证码有效期和请求次数没有进行限制  
  
- token验证之类的，直接将验证内容返回给用户;  
  
- 找回密码功能的进行身份验证内容未加密或者加密算法较弱，容易被猜解，  
  
- 对用户的身份验证在前端进行，导致验证被抓包绕过 ;  
  
- 在最后一步修改密码的动作时，没有校验账号是否通过了验证、短信与手机号是否对应。  
  
任意密码找回的方法  
  
1. 用户凭证暴力破解  
  
2. 返回凭证  
  
3. 邮箱弱token  
  
4. 用户凭证有效性  
  
5. 重新绑定手机和邮箱  
  
6. 服务器验证  
  
7. 用户身份验证  
  
8. 找回步骤  
  
9. 本地验证  
  
10.Token生成可控  
  
11.注册覆盖  
  
12.Session覆盖

**五、常见的逻辑漏洞—交易支付**  
  
  
![](https://mmbiz.qpic.cn/mmbiz_gif/Shiamu7mxKIC6e8e0lygPEVGHsUibo2icpvgCDaKmZ7ic2ic3zDgtSV3NXFvCuMCc2JYoFoweVkuClpEghUxwYxujWks1HTULe2VHdILesrl9qico/640?wx_fmt=gif&from=appmsg "")  
  
  
  
在支付订单时，可以篡改价格为任意金额；或者可以篡改运费或数量为负数，导致总金额降低；  
  
这个漏洞的关键在于服务器没有验证这些数字是否小于0、价格是否从服务器获取、数量是否大于0等。  
  
支付漏洞的主要利用方式  
  
1. 修改支付价格  
  
2. 修改支付状态  
  
3. 修改订单数量  
  
4. 修改附属值（比如优惠券），选择优惠券后修改金额大于等于商品的价格  
  
5. 越权支付，用别人的钱包购买自己的东西  
  
漏洞复现  
  
实际安全中会有一些比较特别的，反正各种能改的参数都去尝试。个数*单价-优惠券个数*单价=总额，每个值都可能存在问题，就看服务自身处理是否有问题。

**六、常见的逻辑漏洞—越权漏洞**  
  
  
![](https://mmbiz.qpic.cn/mmbiz_gif/Shiamu7mxKIC6e8e0lygPEVGHsUibo2icpvgCDaKmZ7ic2ic3zDgtSV3NXFvCuMCc2JYoFoweVkuClpEghUxwYxujWks1HTULe2VHdILesrl9qico/640?wx_fmt=gif&from=appmsg "")  
  
  
  
授权的概念  
：  
  
•   
是对  
特定资源  
的  
读写权限  
。简单的说，就是你的权限能让你做什么事情，而验证则表示是否真的可以对这些资源进行读写。  
  
未授权访问的  
概  
念：  
  
•   
非授权访问是指用户在  
没有通过认证授权  
的情况下，能够  
直接访问需要通过认证才能访问到的页面  
或文本信息。  
  
越权  
漏洞的产生  
  
越权漏洞的成因主要是因为  
开发人员在对数据进行操作  
时  
对客户端请求的数据过分相信，而遗漏了权限的判定。  
  
越权产生的位置  
  
1. 修改、重置、找回其他账户密码  
  
2. 查看、修改其他账户  
未公开的信  
息  
，例如个人资料、文件、数据、程序等  
  
3. 与  
账户  
关联的  
权限操作  
  
![](https://mmbiz.qpic.cn/sz_mmbiz_png/Shiamu7mxKIBpmIVae17aviazcZNLF4PSfckdadYrH3qKpFRIFxibzLWyaUsMYZFaKTgaxHy7eJGKAvt40ia3HkBUDFgPSMLJ64hZIvUVFmqplU/640?wx_fmt=png&from=appmsg "")  
  
越权漏洞的分类  
  
水平越权  
：指  
相同权限  
的  
不同用户  
可以  
互相访问。  
  
•   
举例：两个  
不同的公司  
A和B，通过修改请求，公司A可以任意修改B公司的员工、部门、考勤等信息。  
  
垂直越权  
：指使用  
权限低  
的用户  
可以访问权限较高  
的用户。  
  
•   
举例：  
同一个公司  
不同权限的管理员A和B，通过修改请求，管理员A可以修改不在他管辖范围内的员工、考勤、工资等。

**七、逻辑漏洞的防御**  
  
  
![](https://mmbiz.qpic.cn/mmbiz_gif/Shiamu7mxKIC6e8e0lygPEVGHsUibo2icpvgCDaKmZ7ic2ic3zDgtSV3NXFvCuMCc2JYoFoweVkuClpEghUxwYxujWks1HTULe2VHdILesrl9qico/640?wx_fmt=gif&from=appmsg "")  
  
  
  
逻辑漏洞因为是和业务紧密联系，这种漏洞在被利用时数据包中是没有Payload所言的，所以类似于WAF这种应用漏洞的防御手段显然是没有用的，防御和解决业务逻辑漏洞还是要从业务本身出发，做好风险管理，了解掌握各类业务流程和环节中的关键风险节点和风险程度，以及对应有效的风险管理方法和技术，完善风控系统规则等。简单总结了以下几个方面:  
  
•   
完善权限管理和鉴权机制;  
  
•   
对重要数据进行签名，防止篡改；  
  
•   
重要操作验证在后端进行，比如时间限制，最大值限制等；  
  
•   
加强验证码机制和防爆破、防重放机制；  
  
•   
处理好业务流程顺序执行关联。  
  
除此之外，一切漏洞都产生于代码层面和配置层面，加强应用的上线前安全评估和审计必不可少。
