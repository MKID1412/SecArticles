Linux Kernel是Linux操作系统核心组件，负责进程调度、内存管理、网络通信及硬件资源控制。RDS(Remote Direct Memory Access Socket)是Linux内核中的高性能网络通信协议组件，支持低延迟数据传输，广泛应用于高性能计算、数据库及企业服务器环境。io_uring则是Linux提供的高性能异步I/O框架，用于提升文件与网络I/O处理效率。  
  
2026年5月20日，启明星辰安全应急响应中心（VSRC）监测到Linux Kernel RDS零拷贝双重释放本地权限提升漏洞。该漏洞源于rds_message_zcopy_from_user()函数在zerocopy发送失败路径中，未正确清理op_nents字段，导致页面引用计数处理异常，引发双重释放问题。攻击者可结合io_uring固定缓冲区机制，通过构造失败的RDS zerocopy发送请求，逐步窃取页面引用计数并触发page cache覆盖，最终覆盖SUID-root二进制文件内容，实现本地提权并获取root权限。该漏洞利用链无需额外Capabilities，在启用CONFIG_RDS、CONFIG_RDS_TCP及CONFIG_IO_URING的系统中风险较高，可能导致主机完全失陷。  
## 二、影响范围  
  
该漏洞影响包含RDS zerocopy Tx support功能，且启用CONFIG_RDS、CONFIG_RDS_TCP及CONFIG_IO_URING配置的Linux Kernel版本。公开信息显示，漏洞由提交0cebaccef3ac ("rds: zerocopy Tx support.")引入，具体受影响版本范围暂未在官方公告中明确说明。已知Arch Linux默认启用了相关RDS模块，Fedora部分环境亦可能受影响；Debian与Ubuntu则默认通过blacklist机制限制RDS模块自动加载，可在一定程度上降低漏洞利用风险。此外，系统需满足io_uring_disabled=0且存在可读SUID-root二进制文件等条件，攻击者方可完成完整本地提权利用链。
