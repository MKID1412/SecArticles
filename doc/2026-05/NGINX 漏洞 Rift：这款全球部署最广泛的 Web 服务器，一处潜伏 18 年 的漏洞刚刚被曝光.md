#  NGINX 漏洞 Rift：这款全球部署最广泛的 Web 服务器，一处潜伏 18 年 的漏洞刚刚被曝光

如果你正在运行 **NGINX**  
 服务，从部署概率来看，绝大多数企业与站点都在使用，本周爆出的这条安全新闻值得高度重视。  
  
深度安全研究团队 depthfirst 披露了 **NGINX 开源版与商业版（NGINX Plus）**  
 一处**高危堆缓冲区溢出漏洞**  
，该漏洞已在代码库中**潜伏长达 18 年**  
未被发现。漏洞编号 **CVE-2026-42945**  
，CVSS v4 评分高达 **9.2**  
，被命名为 **NGINX Rift**  
，其影响范围远非普通常规补丁可比。  
  
NGINX 支撑着互联网大量业务场景，广泛用作反向代理、负载均衡、集群入口控制器、应用交付平台，攻击面极大。该漏洞存在于 **ngx_http_rewrite_module（HTTP 重写模块）**  
，所有标准编译版本均默认内置；且触发条件是一种十分常见的配置写法，现实中大量在线业务部署都可能已受影响却浑然不知。  
  
漏洞根源：NGINX 在处理**重写指令**  
时，若配置中同时满足：使用 **PCRE 匿名捕获组**  
（常见 $1  
、$2  
 写法）、替换字符串包含**问号**  
，且同作用域下紧随另一条 rewrite、if 或 set 指令，就会触发缺陷。  
  
漏洞底层原理十分隐晦，但危害后果极其严重：当替换字符串中出现问号时，脚本引擎内部标记位会被置位且**永远不会清空**  
。后续长度计算会启用全新子引擎，不做 URI 转义处理，仅按原始字节大小分配缓冲区；但实际写入操作仍在保留转义标记的原引擎中执行，+  
、%  
、&  
 等字符在拷贝过程中会**单字符扩容为两字节**  
，最终造成**确定性堆缓冲区越界写入**  
，攻击者可通过构造 URI 精准控制内存破坏布局。  
  
depthfirst 发布的报告指出：**一旦存在该类配置写法，NGINX 会按一套转义规则计算目标缓冲区大小，却用另一套规则执行写入，最终造成缓冲区越界、确定性内存损坏。**  
只要运行受影响版本且采用该配置模式的 NGINX，在打补丁或修改配置前均处于暴露风险状态。  
  
这一点尤为关键：不同于多数内存损坏漏洞受系统随机状态影响难以稳定利用，该漏洞越界写入的字节**完全由攻击者请求可控构造**  
，内存破坏可定向塑造、而非随机错乱，大幅提升漏洞利用成功率。  
  
报告补充：**攻击者只需通过 HTTP 访问目标 NGINX 服务，发送单次恶意请求，即可触发工作进程堆溢出，实现远程代码执行。**  
 无需身份认证、无需前置权限、无需已有会话。  
  
在**未开启 ASLR**  
 的环境（部分生产环境仍存在此类配置）下，仅需一条特制 HTTP 请求，就能直接在 NGINX 工作进程中实现远程代码执行；即便开启 ASLR，重复发送恶意请求也可使工作进程陷入**崩溃死循环**  
，导致该实例承载的所有业务站点服务可用性下降。  
  
depthfirst 表示：**越界写入的内容完全取自攻击者构造的 URI，内存破坏可被攻击者精准操控，而非随机触发。**  
 重复请求还能持续打崩工作进程、拖累整站服务可用性。  
  
该漏洞影响 F5 及 NGINX 全系多款产品：NGINX 开源版 **0.6.27 ~ 1.30.0**  
 全版本受影响；NGINX Plus **R32 ~ R36**  
 版本受波及。同时波及：NGINX 实例管理器、F5 WAF for NGINX、NGINX 应用防护 WAF、F5/NGINX DoS 防护组件、NGINX 网关编排组件、NGINX 入口控制器等多个版本分支。**BIG-IP、F5OS、F5 分布式云**  
 产品不受此漏洞影响。  
  
该漏洞于 **2026 年 4 月 21 日**  
 经负责任披露流程同步推送修复版本：  
- NGINX 开源版升级至 **1.30.1 或 1.31.0**  
  
- NGINX Plus R36 安装补丁 **R36 P4**  
，R32 安装 **R32 P6**  
  
- 实例管理器、应用防护、网关组件、入口控制器均已发布对应修复分支版本  
  
本次随 NGINX Rift 漏洞同步修补的还有另外 3 个 NGINX 高危漏洞：  
1. **CVE-2026-42946（CVSS v4 8.3）**  
SCGI/UWSGI 模块存在**内存过度分配漏洞**  
。中间人攻击者篡改上游服务响应包时，可读取 NGINX 工作进程内存，或触发进程重启，影响 scgi_pass、uwsgi_pass 业务配置。  
  
1. **CVE-2026-40701（CVSS v4 6.3）**  
SSL 模块**释放后重用（UAF）**  
 漏洞。远程无认证攻击者在配置 ssl_verify_client  
 开启 / 可选、且启用 OCSP 时可触发，存在有限数据篡改或进程重启风险。  
  
1. **CVE-2026-42934（CVSS v4 6.3）**  
字符集模块**越界读漏洞**  
。同时配置字符集、源字符集、字符集映射，且反向代理关闭缓冲时，可泄露进程内存数据或造成工作进程重启。  
  
官方建议处置方案十分明确：**直接升级修复版本**  
，安装补丁后重启 NGINX，让工作进程加载修复后的二进制程序。  
  
对于无法立即停机升级的环境，针对 **CVE-2026-42945**  
 可采用**配置层临时缓解方案**  
：该漏洞仅在**PCRE 匿名捕获组 + 替换串带问号**  
 组合时触发，只需把 $1/$2  
 这类匿名捕获改为**命名捕获组**  
，即可禁用漏洞触发路径，且无需业务停机。  
  
示例：高危配置：rewrite ^/users/([0-9]+)$ /profile.php?id=$1 last;改为安全配置：rewrite ^/users/(?<user_id>[0-9]+)$ /profile.php?id=$user_id last;  
  
微小配置改动即可有效防护，为补丁升级争取缓冲时间。  
  
截至漏洞披露时，暂无在野真实攻击利用记录；depthfirst 团队与 F5 协同完成负责任披露，确保公开漏洞信息同步提供修复补丁。但安全窗口期不会长期维持，建议尽快加固修复。
