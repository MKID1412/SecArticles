很多入门学习 Web 安全、做 SRC 挖掘的同学， 目录遍历漏洞属于**简单好测、容易出洞、收录率极高**  
的基础高危漏洞。 不用复杂 Payload，简单改路径符号就能读取服务器敏感配置文件。  
  
今天完整讲解：漏洞原理、挖洞思路、实操复现、标准 SRC 报告模板。  
## 一、漏洞基础认知  
### 漏洞原理  
  
网站前端传入文件路径参数时，**未过滤 \.\./ 路径跳转符号**  
， 攻击者利用 \.\./  
 向上穿越目录，读取服务器任意敏感文件：/etc/passwd  
、网站配置、数据库账号密码、源码备份文件等。  
### 常见特征  
  
URL 里出现：path=  
、file=  
、download=  
、img=  
、url=  
带文件路径参数的接口，基本都要测目录遍历。  
## 二、挖洞选目标思路  
  
日常测站重点关注这些功能：  
- 网站文件下载、附件查看  
  
- 图片加载、文件预览接口  
  
- 日志查看、配置导出模块  
  
- URL 中有 file  
、path  
、dir  
、download  
 参数  
  
只要能控制文件路径，直接构造 \.\./  
 测试。  
## 三、漏洞复现实操步骤  
### 步骤 1：正常访问文件抓包  
  
正常网站预览图片链接：  
```
http://xxx.com/view.php?file=logo.jpg

```  
  
抓包观察 file  
 参数可控。  
### 步骤 2：构造遍历 Payload  
  
修改参数为：  
```
file=../../etc/passwd

```  
### 步骤 3：验证漏洞  
  
请求后服务器直接返回 **etc/passwd 系统用户配置内容**  
， 说明未过滤路径符号，**目录遍历 &amp; 任意文件读取漏洞存在**  
。  
## 四、SRC 标准漏洞报告模板  
### 漏洞标题  
  
某企业文件预览接口存在目录遍历漏洞，可读取服务器任意敏感文件  
### 漏洞等级  
  
高危  
### 漏洞描述  
  
该企业文件预览接口 /view\.php  
 的 file  
 参数未对 \.\./  
 路径跳转符号做过滤与安全校验。 攻击者可构造多级 \.\./  
 穿越网站目录限制，读取服务器任意系统文件、网站配置文件、数据库配置、源码备份等敏感内容。 泄露服务器账号、数据库凭证、业务源码等核心数据，可被用于进一步渗透内网、拿下服务器权限，危害严重。  
### 复现步骤  
1. 访问目标网站文件预览功能，正常加载图片获取 URL 参数；  
  
1. 修改 URL 中 file 参数，构造 payload：\.\./\.\./etc/passwd  
；  
  
1. 发送请求，页面直接返回服务器系统用户配置内容；  
  
1. 可继续遍历读取网站配置、数据库配置等敏感文件，漏洞复现成功。  
  
### 影响范围  
1. 读取服务器系统账号、运维信息，为后续渗透提供信息支撑；  
  
1. 泄露数据库账号密码、网站后台配置，可直接登录管理后台；  
  
1. 下载网站源码、备份文件，进行代码审计挖掘更多漏洞；  
  
1. 严重威胁服务器及整个业务集群安全。  
  
### 修复建议  
1. 后端严格过滤 \.\./  
、\.\.\\  
 等路径跳转字符；  
  
1. 采用白名单机制，仅允许指定文件名与目录访问；  
  
1. 对用户传入路径参数做标准化路径解析，限制在网站固定目录内；  
  
1. 禁止直接使用用户输入拼接文件路径，使用映射 ID 替代真实路径。

## 五、新手挖洞学习忠告  
1. 只要看到 file/path/download/img  
 参数，必测目录遍历；  
  
1. Windows 服务器试 \.\./windows/system32/drivers/etc/hosts  
Linux 服务器试 \.\./\.\./etc/passwd  
  
1. 漏洞很好验证，报告写清可读取配置、源码、数据库信息，更容易定级高危。
